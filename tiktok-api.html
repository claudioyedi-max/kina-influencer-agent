What the App Does

The local app is the AXIS Influencer System. It creates AI/tech creator content through an agent pipeline, waits for human approval, then dispatches approved jobs to platform publishers. TikTok publishing is only attempted after approval.

1. Generate campaign content Trend, research, strategy, script, platform, compliance, and scheduling agents prepare a job.
2. Human approval gate Approval through Telegram or POST /jobs/{id}/approve is the only path to publishing.
3. TikTok draft upload After TikTok approval and backend token setup, the backend uses the Content Posting API to upload approved videos as drafts for the creator to finish in TikTok.

OAuth Redirect

Configure TikTok OAuth to return to a backend endpoint, not to a static browser-only token exchange. The backend exchanges the authorization code and stores tokens securely.

https://your-api-domain.com/api/auth/tiktok/callback

For local testing, use a backend callback such as http://localhost:8000/api/auth/tiktok/callback. Do not put TIKTOK_CLIENT_SECRET on GitHub Pages.

Requested TikTok Scopes

user.info.basic Identifies the connected creator account so Kina can show the account name and confirm the correct user approved the connection.
video.upload Uploads a human-approved video to the creator account as a TikTok draft for final editing and posting inside TikTok.

Secure API Key Use

  • TikTok client key, client secret, access token, refresh token, and open id stay in the private backend environment.
  • The public site can call backend endpoints, but it never stores TikTok secrets.
  • The app can run in dry-run mode while TikTok review is pending.
  • Content is not auto-posted. A creator or operator must approve the job first.
